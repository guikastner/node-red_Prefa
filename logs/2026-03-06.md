# 2026-03-06

## Ajuste de permissões de internet do Node-RED

- Revisado `Agents.md`: requisito explícito de acesso à internet para o container Node-RED.
- Identificada causa do bloqueio: rede principal Docker com `internal = true` e Node-RED conectado apenas a essa rede.
- Atualizado `containers.tf` para conectar `docker_container.node_red` também na rede Docker padrão `bridge`.
- Mantida a ausência de `ports` publicados no host, preservando a restrição de não expor portas.

## Atualização de documentação da rede

- Atualizado `README.md` para refletir o modelo de rede atual.
- Documentado que o Node-RED opera em dupla rede (`${name_prefix}net` interna + `bridge` para saída).
- Documentado que o MongoDB permanece apenas na rede interna.
- Reforçado no README que não há publicação de portas no host.
